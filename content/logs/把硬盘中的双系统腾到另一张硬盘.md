旧的1t固态里面放了一个ubuntu20.4和一个win10，新的硬盘2t

## **迁移 win 系统**

清空2t盘并格式化

用win10的应用`diskgenius`选择源和目标直接迁移，这个工具会自己分引导区，迁移有个选项是设置系统空间大小，设置成自己需要的

## 迁移 Ubuntu 系统

### 迁移系统盘

在2t盘上做好交换空间（建议16G）和系统空间（ext4），

> 踩雷
>
> 这时候不要使用`diskgenius`来做了，因为 1. 他的系统迁移是迁移win；2. 如果使用分区克隆，ubuntu的文件只能用文件复制的方式克隆，会导致 Python 的软链接失效，使用会出很多问题

应该使用 Ubuntu 的 `GParted`  ，在电脑上插个带 ubuntu 镜像的U盘，点 Try Ubuntu ，这个系统里面就有这个工具，打开后选择源分区右键复制，在目标分区粘贴，然后点左上角绿色对勾执行待处理操作，系统盘就可以完好的复制过去。

> 如果在这一步骤发现要用的分区后面带了个钥匙，是因为这个分区挂载到了系统上，查一下挂载位置卸载就行

可以确定 `nvme0n1p1` 是引导区（/boot/efi），Ubuntu 系统盘是 `nvme0n1p5`（这个要看上一步是往哪个分区迁移的 ubuntu 系统）

```bash
# 挂载主分区和 EFI 分区
sudo mount /dev/nvme0n1p5 /mnt
sudo mount /dev/nvme0n1p1 /mnt/boot/efi

# 绑定硬件接口
sudo mount --bind /dev /mnt/dev
sudo mount --bind /proc /mnt/proc
sudo mount --bind /sys /mnt/sys

# 进入迁移后的系统
sudo chroot /mnt

# 到这里你发现用户名啥的变成你这个分区的用户名了
# 把 GRUB 引导注入新硬盘
grub-install /dev/nvme0n1

# 刷新引导并退出
update-grub
exit
```

重启，拔掉刚才插的U盘，华硕主板按 F2 ，联想Y7000 按 F12 , 设置对从哪个硬盘引导（如果你多个硬盘都有系统），选择Ubuntu

### 启动优化

#### 去掉开机动画显示开机加载日志输出

联想Y7000 Ubuntu开机不会展示命令行输出而是一个没什么用的开机动画，把他去掉

```bash
sudo nano /etc/default/grub
```

找到 `GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"`，里面的 `quiet splash` 删掉

#### 增加 Win 的启动项

还是刚才的  `GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"` ，刚才删掉后面变成了“”，在引号里面加上 `GRUB_DISABLE_OS_PROBER=false` 

#### 刷新配置

```bash
sudo update-grub
```

注意刷新之前把不用的存储设备全部拔掉，不然会在 grub 界面里一直有这些存储设备里的系统的选项

### 设置Swap并开启断电休眠恢复

用 `GParted` 在磁盘未分配空间处或者刚才分好的16G分区新建一个文件系统 `linux-swap` 的 16GB 分区。记住分区名字（我的是 `nvme0n1p6`）

```bash
# 关闭旧的临时 Swap（如果有）
sudo swapoff -a

# 格式化并启用
sudo mkswap /dev/nvme0n1p6
sudo swapon /dev/nvme0n1p6
```

再用 `lsblk -f` 获取 swap 的 UUID ，写入 `/etc/fstab` 

```
UUID=查到的 none swap sw 0 0
```

启动断电休眠

```bash
sudo nano /etc/initramfs-tools/conf.d/resume
# 写入
RESUME=UUID=新Swap_UUID

sudo nano /etc/default/grub
# 写入
GRUB_CMDLINE_LINUX_DEFAULT="resume=UUID=新Swap_UUID"

# 更新配置
sudo update-initramfs -u
sudo update-grub
```

现在再重启，就能正常使用了
